# 1.1.0 Release Notes

## New features
- Add IP-aware column mapping flags: `-cidr-ip-col` and `-cidr-ip-cidr-col` with case-sensitive matching.
- Add timestamped batch output contract:
  - `scan_results-YYYYMMDDTHHMMSSZ[-n].csv`
  - `opened_results-YYYYMMDDTHHMMSSZ[-n].csv`
- Keep output pair sequence aligned on same-second collision (`-n` shared by both files).
- Enforce resume path rule:
  - explicit `-resume` uses the same file for load/save
  - default fallback is `<output-dir>/resume_state.json`
- Add structured observability events with required fields:
  - `target`, `port`, `state_transition`, `error_cause`
- Add progress and completion summary events for long scans.

## Bug fixes
- Strengthen pressure API failure escalation behavior and persist resume state on fatal interruption paths.
- Normalize output schema behavior for missing metadata values (`fab_name`, `cidr_name` -> empty string).

## Breaking changes
- Output filenames are now timestamped batch names by default instead of writing to a fixed filename.

## Compatibility and migration notes
- If existing automation parsed fixed filenames (`scan_results.csv` / `opened_results.csv`), update it to glob by batch pattern (`scan_results-*.csv`, `opened_results-*.csv`).
- If your workflow depends on resume location, pass `-resume <path>` explicitly to pin file location.
- Existing CSV header order remains unchanged:
  `ip,ip_cidr,port,status,response_time_ms,fab_name,cidr_name`.
