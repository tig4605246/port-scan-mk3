# 1.2.0 Release Notes

## New features
- Add rich input parsing for 10 canonical columns:
  `src_ip, src_network_segment, dst_ip, dst_network_segment, service_label, protocol, port, decision, policy_id, reason`.
- Add canonical-header matching rules that ignore case and surrounding whitespace but reject alias
  headers.
- Add row-level validation with mixed valid/invalid row handling and explicit validation error
  reporting.
- Add execution-key deduplication in the scan pipeline using `dst_ip:port/protocol`.
- Refactor CLI and scan orchestration into focused collaborators:
  `cmd/port-scan/command_handlers.go`, `pkg/scanapp/input_loader.go`,
  `pkg/scanapp/runtime_builder.go`, `pkg/scanapp/task_dispatcher.go`,
  `pkg/scanapp/pressure_monitor.go`, `pkg/scanapp/result_aggregator.go`, and
  `pkg/scanapp/resume_manager.go`.
- Record branch-level red/green/refactor evidence in
  `specs/001-solid-refactor-tdd/quickstart.md` and keep full-repo verification gates green.

## Bug fixes
- Reduce coupling between `cmd/port-scan/main.go` and scan runtime behavior so CLI routing changes
  no longer require touching orchestration internals.
- Harden regression coverage for progress reporting, completion summaries, resume persistence, and
  cross-story pressure/resume scenarios.
- Keep `validate` output payloads, timestamped scan artifact naming, and resume save behavior stable
  during the SOLID refactor.

## Breaking changes
- Output schema now appends these fields after the existing prefix:
  `service_label, decision, policy_id, reason, execution_key, src_ip, src_network_segment`.

## Compatibility and migration notes
- Existing consumers that read only the historical output prefix remain compatible because the
  original field order is preserved at the front of each row.
- If downstream tooling validates exact header sets, update it for the expanded output schema.
- CLI flags remain backward compatible; no migration is required for `scan` or `validate`
  invocation patterns.
- Inputs that rely on alias headers such as `source_ip` are now rejected; rename them to the
  canonical column names before running `validate` or `scan`.
