# Release 1.2.0

## Features

- 新增 rich input parser，支援 10 個 canonical 欄位：
  `src_ip, src_network_segment, dst_ip, dst_network_segment, service_label, protocol, port, decision, policy_id, reason`。
- 新增欄位匹配規則：僅接受 canonical 名稱，匹配時忽略大小寫與前後空白，不接受 alias。
- 新增列級驗證與錯誤碼輸出，支援 mixed valid/invalid row 處理。
- 掃描執行層新增 execution-key 去重：`dst_ip:port/protocol`。

## Refactor & Governance

- 依據憲章新增的 SOLID 結構原則，將 CLI 與 scan orchestration 拆分為明確 collaborator：
  `cmd/port-scan/command_handlers.go`、`pkg/scanapp/input_loader.go`、
  `pkg/scanapp/runtime_builder.go`、`pkg/scanapp/task_dispatcher.go`、
  `pkg/scanapp/pressure_monitor.go`、`pkg/scanapp/result_aggregator.go`、
  `pkg/scanapp/resume_manager.go`。
- `cmd/port-scan/main.go` 現在只保留 process entry 與 top-level routing；
  `pkg/scanapp/scan.go` 收斂為 orchestration glue，不再直接承擔輸入載入、dispatch、
  壓力監控、結果彙整與 resume persistence。
- 這次重構遵循嚴格 TDD 證據鏈，red/green/refactor 命令已記錄於
  `specs/001-solid-refactor-tdd/quickstart.md`。

## Contract Updates

- writer 輸出欄位擴展為：
  `ip, ip_cidr, port, status, response_time_ms, fab_name, cidr_name, service_label, decision, policy_id, reason, execution_key, src_ip, src_network_segment`。
- 既有前置欄位順序維持不變，舊 consumer 可沿用前綴欄位。
