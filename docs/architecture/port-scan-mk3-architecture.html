輸入：CIDR CSV、Port Input；輸出：scan/opened CSV、resume_state、runtime events。

P03-Sad-Path-Error-Recovery

Diagram Preview

三條失敗路徑：
A. validation fail -> early exit
B. pressure API repeated failure -> escalation -> save resume -> non-zero exit
C. cancel/signal -> save resume -> rerun with -resume -> recovered completion

mxGraph XML

Mapping Notes

  • 每條分支包含 trigger、effect、recovery。
  • Sad path B 與 C 都要求落地 resume_state。
  • 同頁保留 TCP 探測邊界語意，避免誤解為 RST 強制策略。

Validation Checklist

  • 三類 sad path 全部存在
  • resume recovery 流程完整
  • 非 RST 強制斷線聲明存在
